# UX-2008 destructive action 缺少确认或撤销路径

## 快速信息

| 名称     | 说明          |
| -------- | ------------- |
| 前缀     | UX            |
| 范围     | 用户体验      |
| 严重程度 | `warning`     |
| 阶段     | `interaction` |
| 可重试   | 否            |

## 含义

UX-2008 表示 destructive action 缺少确认或撤销路径。请先确认当前页面、项目状态和最近操作，再按建议操作处理。

## 触发条件

删除、覆盖、清空、发布或不可逆变更没有确认、预览、撤销或恢复机制

## 建议操作

添加确认、撤销、回收站、版本回退或明确风险提示
