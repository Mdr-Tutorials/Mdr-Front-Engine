# COD-2016 类型推断超过复杂度上限

## 快速信息

| 名称     | 说明      |
| -------- | --------- |
| 前缀     | COD       |
| 范围     | 用户代码  |
| 严重程度 | `warning` |
| 阶段     | `symbol`  |
| 可重试   | 是        |

## 含义

COD-2016 表示 类型推断超过复杂度上限。请先确认当前页面、项目状态和最近操作，再按建议操作处理。

## 触发条件

表达式、泛型、条件类型、递归类型或大型 union 导致语言服务或作者态类型推断超过时间、深度或内存上限

## 建议操作

拆分表达式，添加显式类型，或减少泛型递归复杂度

## 上报时提供

- 错误码和 requestId
- 当前项目或工作区 ID
- 触发该错误的操作
- 可复现时的最小文档或配置
